# 表情語音點餐機器人專案規格

## 專案概念

本專案是一個「表情機器人 + iPad 菜單螢幕 + 全語音點餐」的點餐系統。

使用者主要透過語音操作，螢幕則作為確認工具。整體體驗應該像是在跟一個可愛、會引導點餐的表情角色互動：使用者看菜單、直接說想點什麼、看螢幕確認訂單，最後用語音確認送出。

核心設計原則：

> 螢幕是確認工具，語音是操作工具。

使用者不需要按按鈕，理想流程是：

1. 看螢幕
2. 說餐點
3. 看訂單
4. 說確認

## 目標體驗

### 核心流程

```text
待機歡迎
→ 偵測有人靠近
→ 表情機器人開口引導
→ 使用者語音點餐
→ 螢幕即時顯示訂單
→ 表情機器人逐步確認
→ 使用者語音確認
→ 訂單完成
→ 表情機器人送出 / 感謝
```

### 主要互動角色

- 表情角色：負責歡迎、引導、回覆、確認與感謝。第一版 App 需保留貓咪版與柯基版兩種角色設計，並支援用設定或 build config 切換。
- iPad 螢幕：顯示菜單、目前訂單、候選選項、最終確認與訂單完成畫面。
- 使用者：只需要看螢幕並用語音點餐。
- 後台 / POS / 廚房：接收已確認的訂單。

## 第一版最小功能

第一版不要做太複雜，先支援 5 類語音操作：

1. 新增餐點
2. 修改餐點
3. 刪除餐點
4. 完成點餐
5. 確認送出

第一版重點不是完整處理所有自然語言，而是建立一個穩定、可確認、不亂下單的語音點餐閉環。

## 技術選項

### 開發與執行環境

- 開發環境：macOS。
- 主要 App 技術：SwiftUI。
- 目標執行裝置：iPad。
- 第一版輸入方式：iPad 麥克風錄音。
- 第一版語音辨識：使用 iOS 內建語音辨識能力。
- 角色設定：保留 cat / corgi 兩套表情角色，可透過 App 設定或 build config 選擇預設角色。

第一版先不要導入複雜外部語音服務。目標是先確認完整點餐流程可以在 iPad 上跑通，包含錄音、語音轉文字、意圖解析、螢幕更新與最終確認。

### 語音辨識策略

第一版使用 iOS 內建 Speech framework 作為主要語音辨識方案。

基本需求：

- 可以請求麥克風權限。
- 可以請求語音辨識權限。
- 可以從 iPad 麥克風收音並轉成文字。
- 可以把辨識結果交給點餐意圖解析流程。
- 可以處理沒有聲音、聽不清楚、權限被拒絕等狀況。

### CLI 測試策略

除了 iPad App，也需要能用 CLI 測試語音解析流程。

第一版 CLI 測試目標：

- 可以輸入 `.wav` 檔作為測試音訊。
- 可以對 `.wav` 進行語音辨識。
- 可以輸出辨識文字。
- 可以把辨識文字丟進同一套點餐意圖解析邏輯。
- 可以輸出解析後的訂單操作，例如新增、修改、刪除、完成點餐或確認送出。

CLI 測試範例概念：

```text
orderbot transcribe tests/audio/add_chicken_bento.wav
→ 我要一份雞腿便當，不要辣

orderbot parse tests/audio/add_chicken_bento.wav
→ intent: add_item
→ item: 雞腿便當
→ quantity: 1
→ note: 不要辣
```

CLI 與 iPad App 應盡量共用核心邏輯：

- 語音辨識輸出文字後，進入相同的意圖解析流程。
- 訂單狀態更新使用相同的資料模型。
- 測試案例可以先用文字測試，再用 `.wav` 檔測試語音辨識結果。

## 狀態機

建議狀態如下：

```text
Idle 待機
↓
Greeting 歡迎
↓
Listening 聽使用者點餐
↓
Parsing 解析語音
↓
OrderUpdated 更新訂單
↓
AskMore 詢問是否繼續
↓
FinalConfirm 最終確認
↓
Completed 完成送單
```

錯誤與例外狀態：

- `Unclear`：語音辨識或餐點解析不清楚。
- `NoSpeech`：沒有聽到使用者說話。
- `Cancel`：使用者取消目前操作。
- `Reset`：使用者要求全部重來。
- `HumanHelp`：請店員協助。

## 使用流程細節

### 1. Idle：待機狀態

螢幕顯示表情角色與歡迎文案：

```text
😺 歡迎光臨
請看菜單，想點什麼可以直接跟我說喔
```

表情角色可以偶爾：

- 眨眼
- 搖尾巴
- 看向使用者

可選語音：

```text
歡迎光臨～想點餐可以直接跟我說喔。
```

觸發方式：

- 人靠近
- 鏡頭偵測
- 距離感測器
- 按開始按鈕作為備援

### 2. Greeting：開始點餐

使用者靠近後，表情機器人說：

```text
你好，我是點餐助手。請看菜單，直接說你想點的餐點就可以了。
```

螢幕顯示語音範例：

```text
請直接說：
「我要一份雞腿便當，不要辣，再加一杯紅茶」
```

接著進入 `Listening` 狀態。

### 3. Listening / Parsing：使用者點餐

使用者說：

```text
我要一份雞腿便當，不要辣，再加一杯紅茶。
```

系統流程：

```text
語音辨識
→ 解析菜名、數量、備註
→ 螢幕更新
→ 表情機器人簡短回覆
```

螢幕顯示：

```text
目前已點：

1. 雞腿便當 x1
   備註：不要辣

2. 紅茶 x1

請繼續說，或說「好了」完成點餐
```

角色語音：

```text
好的，已加入雞腿便當一份，不要辣，還有紅茶一杯。還需要其他的嗎？
```

### 4. OrderUpdated / AskMore：繼續加點

使用者說：

```text
再加一份薯條。
```

螢幕更新：

```text
目前已點：

1. 雞腿便當 x1
   備註：不要辣

2. 紅茶 x1

3. 薯條 x1
```

表情機器人說：

```text
好的，已加入薯條一份。
```

### 5. 修改訂單

使用者可以直接說：

```text
紅茶改半糖少冰
雞腿便當改成兩份
薯條不要了
取消紅茶
全部重來
```

例如使用者說：

```text
紅茶改半糖少冰。
```

螢幕更新：

```text
2. 紅茶 x1
   甜度：半糖
   冰塊：少冰
```

表情機器人說：

```text
好的，紅茶改成半糖少冰。
```

### 6. Unclear：不確定時的確認

如果系統不確定使用者要點什麼，不能直接下單，必須請使用者確認。

例如辨識到：

```text
我要雞腿……
```

螢幕顯示候選：

```text
我聽到可能是：

A. 雞腿便當
B. 雞排便當
C. 雞肉飯

請說：「第一個」、「第二個」或「第三個」
```

表情機器人說：

```text
我有點不確定，你是要雞腿便當、雞排便當，還是雞肉飯？
```

使用者說：

```text
第一個。
```

表情機器人說：

```text
好的，雞腿便當一份。
```

## 完成點餐

### 進入 FinalConfirm 的語音

使用者可以說：

```text
好了
就這樣
不用了
送出
結帳
完成
```

系統進入 `FinalConfirm` 狀態。

表情機器人說：

```text
我幫你確認一下，你點的是：雞腿便當一份，不要辣，紅茶一杯，半糖少冰，薯條一份。正確嗎？
```

螢幕顯示大字版：

```text
請確認訂單

1. 雞腿便當 x1
   不要辣

2. 紅茶 x1
   半糖 / 少冰

3. 薯條 x1

請說：
「正確」送出
「修改」繼續修改
「取消」重新點餐
```

### 訂單完成條件

只有在使用者明確說出確認語，才算完成訂單。

確認語包含：

```text
正確
對
沒錯
可以
送出
確認
```

完成後流程：

```text
訂單鎖定
→ 送到後台 / POS / 廚房
→ 顯示訂單編號
→ 表情機器人感謝
```

表情機器人說：

```text
好的，訂單已送出。你的號碼是 23 號，謝謝你～
```

螢幕顯示：

```text
訂單完成

號碼：23

請稍候取餐
```

## 語音腳本範例

### 歡迎

```text
歡迎光臨～請看菜單，想吃什麼可以直接跟我說。
```

### 聽不清楚

```text
抱歉，我剛剛沒有聽清楚，可以再說一次嗎？
```

### 加入餐點

```text
好的，已加入雞腿便當一份。
```

### 詢問繼續

```text
還需要其他的嗎？如果點好了，可以說「好了」。
```

### 最終確認

```text
我幫你確認一下目前的餐點，正確的話請說「確認」。
```

### 完成

```text
訂單已送出，謝謝你～
```

## 第一版語音意圖

### 1. 新增餐點

範例：

```text
我要一份雞腿便當
再加一杯紅茶
幫我加一份薯條
```

預期行為：

- 解析餐點名稱、數量與備註。
- 加入目前訂單。
- 螢幕即時更新。
- 表情機器人用一句話確認已加入。

### 2. 修改餐點

範例：

```text
紅茶改半糖少冰
雞腿便當改成兩份
不要辣改成小辣
```

預期行為：

- 找到既有訂單項目。
- 更新數量、客製選項或備註。
- 若找不到明確項目，進入確認或不清楚流程。

### 3. 刪除餐點

範例：

```text
薯條不要了
取消紅茶
幫我刪掉雞腿便當
```

預期行為：

- 從目前訂單移除指定項目。
- 螢幕更新。
- 表情機器人確認已刪除。

### 4. 完成點餐

範例：

```text
好了
就這樣
不用了
結帳
完成
```

預期行為：

- 不直接送出訂單。
- 進入最終確認狀態。
- 大字顯示完整訂單。
- 請使用者明確確認。

### 5. 確認送出

範例：

```text
正確
對
沒錯
可以
送出
確認
```

預期行為：

- 鎖定訂單。
- 送出到後台 / POS / 廚房。
- 顯示訂單號碼。
- 表情機器人感謝使用者。

## 設計限制與原則

- 不確定時不能亂下單，必須請使用者確認。
- 完成點餐與確認送出要分成兩步。
- 螢幕內容要大字、清楚、可快速確認。
- 語音回覆要短，避免讓使用者等待太久。
- 使用者不應被要求學習複雜指令。
- 第一版先完成穩定閉環，再擴充更多自然語言、推薦餐點、付款與會員功能。
